Java中的自动装箱和自动拆箱都会增加额外的内存和cpu资源的消耗，为了减小基本类型和包装类型之间的转换，
Java8中提供的Stream类的某些方法对基本类型和装箱类型做了区分。
Java8中仅对整型、长整型和双浮点型做了特殊处理(Integer、Double和Long)，因为它们在数值计算中用得最多，特殊处理后的系统性能提升效果最明显。

对基本类型做特殊处理的方法在命名上有明确的规范：
1)、如果方法返回类型为基本类型，则在基本类型前加To，如ToLongFunction.
    T ---> ToLongFunction ---> Long
2)、如果参数类型为基本类型，则不加前缀只加类型名即可，如LongFunction.
    long ---> LongFunciton ---> T
3)、如果高阶函数使用基本类型，则在操作后加后缀To再加基本类型，如mapToLong.

4)、这些基本类型都有与之对应的Stream，以基本类型名为前缀，如LongStream


重载解析：
Java中可以重载方法，造成多个方法有相同的方法名，但签名不一样。这在推断类型时会带来问题，因为可能会推断出多种类型。
这时javac会挑出最具体的类型。

Lambda表达式作为参数时，其类型由它的目标类型推导得出，推导过程遵循如下规则：
1)、如果只有一个可能的目标类型，由相应函数接口里的参数类型推导得出。
2)、如果有多个可能的目标类型，由最具体的类型推导得出。
3)、如果有多个可能的目标类型且最具体的类型不明确，则需人为指定类型。


默认方法：
默认方法是添加在类库中新增加的方法之上，此举可避免第三库因为继承或者实现该类库需要重新实现新增方法，使得Java能够向后兼容。

默认方法成了虚方法--和静态方法刚好相反。任何时候一旦与类中的方法产生冲突，都优先选择类中定义的方法。
与接口中定义的默认方法相比，类中重写的方法更具体。(类中重写的方法优先级高于接口中定义的默认方法)


多重继承：
接口允许多重继承，因此有可能碰到两个接口包含签名相同的默认方法的情况。
