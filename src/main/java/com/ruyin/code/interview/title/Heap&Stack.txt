heap与stack的区别：
1、stack和heap都是Java用来在ram中存放数据的地方。
2、stack的优势在于存取速度比heap快；heap的优势是可以动态地分配内存大小。
3、Java中的数据类型有两种:基本类型数据和包装类数据。
    基本类型数据定义在栈中，栈中的数据在栈中是共享的，在栈中定义的两个变量如果字面值一样，则不会创建一个新的字面值让变量去引用，而是将其指向原字面值的引用地址。
    另外字面值的引用与类对象的引用不同，若两个类对象的引用同时指向一个对象，如果一个对象引用变量修改了这个对象的内部状态，那么另一个对象引用变量也立即反映出这个变化。
    相反，通过字面值的引用来修改字面值则不会导致另一个指向此字面值的引用的值跟着改变，本质在于改变的值首先查看栈中是否存在该字面值，存在，指向，不存在，新建一个新的字面值并指向它。

4、每个JVM的线程都有自己的私有栈空间，随线程创建而创建，Java的stack存放的是frames，Java的stack与c不同，只是存放本地变量、返回值和调用方法，不允许直接push和pop frames，
因为frames可能是有heap分配的，所以Java的stack分配的内存不需要是连续的。Java的heap是所有线程共享的，heap存放所有的runtime data，里面是所有的对象实例和数组，heap是JVM启动时创建的。
堆栈中的信息不共享，堆中的信息不共享。

5、数据类型包装类的值不可修改，不仅仅是String类的值不可修改，所有的数据类型包装类都不能更改其内部的值。
